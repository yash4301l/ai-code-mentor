# Submission Ready Checklist

Fill this before uploading to dashboard.

## 1) Project PPT (Step-1 Filter)
- [ ] Deck title is clear in first 5 seconds.
- [ ] Problem, solution, architecture, demo proof, and impact are included.
- [ ] Last slide has all mandatory links.
- [ ] PPT filename is clean: `AI-Code-Mentor_Hackathon-Pitch.pptx`.

## 2) GitHub Repository
- [ ] Repo URL: `[PASTE_GITHUB_URL]`
- [ ] Repo is public OR evaluator access granted.
- [ ] `README.md` includes setup steps and architecture image.
- [ ] No broken links or missing folders.

## 3) Working Prototype Link
- [ ] Live URL: `[PASTE_LIVE_URL]`
- [ ] Opens without login for evaluators.
- [ ] Backend API responds for at least one demo case.
- [ ] Add one-click demo instruction near top of app.

## 4) Demo Video
- [ ] Video URL: `[PASTE_VIDEO_URL]`
- [ ] Length: 2 to 4 minutes.
- [ ] Includes: problem -> product flow -> blocked hallucination example -> impact.
- [ ] Thumbnail/title clearly says project name.

## 5) Project Summary
- [ ] Summary link or text location: `[PASTE_SUMMARY_LINK_OR_FILE]`
- [ ] Includes: problem, solution, innovation, stack, impact, roadmap.
- [ ] Keep first paragraph high-impact and non-generic.

## Final QA (Do This Once)
- [ ] All 5 links open in incognito mode.
- [ ] Same project name used everywhere (PPT, repo, video, summary).
- [ ] Contact details/team details included in PPT final slide.
- [ ] Submission done at least 2 hours before deadline.
